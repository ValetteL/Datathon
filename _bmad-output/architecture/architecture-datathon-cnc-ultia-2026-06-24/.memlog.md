- (constraint) Stack Python + LangChain/LangGraph décidée par l'équipe avant le datathon
- (constraint) Environnement d'exécution : Google Colab partagé — pas d'infra locale
- (constraint) Durée : 3 jours (J1=comprendre, J2=prototyper, J3=orchestrer)
- (constraint) Équipe : 6 personnes, rôles fixes (DA×2, AgentDev×2, Infra×1, PM×1)
- (constraint) Corpus : data.xlsx, 30 colonnes, plage 26 mars → fin avril 2026

- (constraint) Règle éditoriale : ton neutre, décrire avant de juger, analyse ≠ débat
- (constraint) Livrable final : démo live + pitch 10 min devant jury (J3)
- (version) Python ≥ 3.10 (Google Colab 2026 default)
- (version) langchain ≥ 0.3, langgraph ≥ 0.2 (pip install langchain langgraph pandas)
- (version) pandas ≥ 2.0 pour lecture data.xlsx (openpyxl en backend)
- (version) LLM cible : Gemini 2.0 Flash via Google AI Studio (gratuit, generous quota)
- (decision) AD-1 | PARADIGME : Pipeline LangGraph StateGraph comme orchestrateur central. Binds: tout agent est un nœud dans le même graphe; le graphe est la seule entité qui connaît l'ordre d'exécution. Prevents: appels directs agent-à-agent hors du graphe; duplication de l'état.
- (decision) AD-2 | ÉTAT PARTAGÉ : TypedDict CrisisState unique transmis entre tous les nœuds. Champs: raw_df, tweets_sample, narratives, actors, alerts, strategy_options, draft_response, human_approved. Binds: chaque agent lit et écrit UNIQUEMENT dans ses champs déclarés. Prevents: agents qui se passent des données via des variables globales ou des effets de bord.
- (decision) AD-3 | ACCÈS DONNÉES : Une seule fonction d'outil load_corpus(path) → DataFrame retourne le df nettoyé. Les agents reçoivent des slices pandas (filter + sample), jamais le fichier brut. Binds: les transformations de nettoyage (types, doublons, NaN) sont centralisées dans load_corpus. Prevents: chaque agent qui ferait sa propre lecture et son propre nettoyage.
- (decision) AD-4 | OUTPUTS TYPÉS : Chaque agent produit un objet Pydantic (NarrativeResult, AlertSignal, StrategyOptions, DraftCommunique). Binds: le schéma Pydantic est le contrat d'interface entre agents. Prevents: agent suivant qui interprèterait un output textuel libre différemment selon le modèle utilisé.
- (decision) AD-5 | NEUTRALITÉ : Un module prompts.py centralise le system prompt de neutralité (décrire sans juger, ton factuel) importé par TOUS les agents. Binds: les agents ne peuvent pas définir leur propre system prompt sans passer par ce module. Prevents: un agent Rédacteur qui prendrait une posture partisane dans ses drafts.
- (decision) AD-6 | HUMAN GATE : Un nœud HumanGate (interrupt) s'intercale entre AgentVeille et AgentStratège. L'orchestrateur suspend, présente l'analyse, attend une confirmation humaine, puis reprend. Binds: le Stratège ne tourne jamais sans validation préalable de l'analyse. Prevents: génération de réponses institutionnelles basées sur une analyse hallucinée.
- (decision) AD-7 | LLM PROVIDER : ChatGoogleGenerativeAI (Gemini 2.0 Flash) comme LLM par défaut, injectable via variable d'environnement GOOGLE_API_KEY. Binds: tous les agents instancient le LLM via une factory get_llm() unique. Prevents: agents qui hardcoded un provider différent; clé API en clair dans le code.
- (decision) AD-8 | SÉQUENCE PIPELINE : Analyste → Veille → [HumanGate] → Stratège → Rédacteur. Pas de parallélisme J1-J2 (corpus trop petit, Colab mono-thread suffisant). Prevents: AgentVeille qui tournerait sans les narratifs de l'Analyste; AgentStratège qui répondrait sans les alertes.
- (decision) AD-9 | GENERALISATION : Les agents reçoivent un paramètre corpus_config (dict: événement, période, mots-clés déclencheurs) injecté à l'init du graph. Binds: aucune référence hardcodée à 'Ultia', 'CNC', ou 'mars 2026' dans le code agent. Prevents: pipeline non-réutilisable pour un autre cas de crise.
- (decision) AD-10 | ANTI-HALLUCINATION : Chaque output Pydantic inclut un champ source_tweet_ids: list[str] obligatoire. L'agent ne peut émettre un claim sans citer les postID sources. Binds: le Rédacteur ne reformule que des claims avec source_tweet_ids renseignés. Prevents: affirmations inventées non traçables dans le rapport final.
