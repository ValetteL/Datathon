- (technique) started First Principles Thinking — déconstruire chaque hypothèse jusqu'aux faits bruts, reconstruire depuis zéro
- (idea by coach) Une crise virale est structurellement : signal → amplification → feedback loop → conséquence institutionnelle — partir de là, pas de ce qu'on croit savoir
- (idea by coach) Séparer le détecteur de crise (est-ce une crise ?) du pipeline d'analyse (quel type, quelle dynamique) — deux jobs distincts
- (idea by coach) Construire la couche de vérification EN PREMIER avant tous les agents — c'est le socle de confiance, pas un ajout
- (idea by coach) L'orchestrateur doit être STATEFUL et tracer le cycle de vie : Émergence → Pic → Déclin — chaque phase active des agents différents
- (idea by coach) L'extraction des faits est un problème d'extraction de données structurées (pas de résumé) — timestamps, claims, acteurs, IDs de posts
- (idea by coach) Le mapping narratif est fondamentalement un problème de clustering : regrouper les tweets par similarité sémantique puis labelliser chaque cluster
- (idea by coach) La propagation est un problème de traversée de graphe (chaînes RT, threads replies, QTs) — pas un problème NLP
- (idea by coach) La détection de coordination est un problème d'anomalie temporelle : synchronicité de posting, formulations identiques
- (idea by coach) Chaque output d'agent doit être un JSON structuré (Pydantic), jamais de prose brute — la prose arrive en dernier avec l'agent rédacteur
- (idea by coach) La phase réponse requiert le contexte institutionnel en input : quel est le mandat, quels sont les risques propres à CETTE institution
- (idea by coach) Human-in-the-loop non optionnel pour la réponse : le système recommande, les humains décident — l'agent ne publie jamais seul
- (idea by coach) Deux modes d'exécution : temps-réel (stream processing, faible latence) et post-mortem (batch, haute qualité) — même interface, pipeline distinct
- (idea by coach) Les 5 axes ne sont pas indépendants — construire un graphe de dépendances : acteurs d'abord, puis propagation, puis coordination
- (idea by coach) Le 'pourquoi ça a pris' est une question de réseau et d'effet de réseau, pas de contenu — ne pas chercher dans le texte ce qui est dans la topologie
- (idea by coach) Le time-to-peak est une métrique clé : crises <2h vs >24h nécessitent des stratégies de réponse radicalement différentes
- (idea by coach) Le corpus du cas Twitch = jeu de données étiqueté de référence pour évaluer et calibrer tout le système
- (technique) started Mind Mapping — ramification depuis le nœud central vers tous les sous-domaines
- (idea by coach) Agent Timeline : reconstruire la séquence factuelle horodatée AVANT toute interprétation — acte fondateur du pipeline
- (idea by coach) Agent Acteur : classer chaque compte en rôles (initiateur / amplificateur / contre-voix / média / institution / bot suspect)
- (idea by coach) Utiliser un algorithme type PageRank sur le graphe de diffusion pour trouver les nœuds les plus influents — pondérer par followers × vitesse
- (idea by coach) Indice de compétition narrative : mesurer quelle narrative gagne/perd du terrain heure par heure (part de voix par cluster)
- (idea by coach) Agent contenu visuel : les crises sont souvent propulsées par des screenshots et mèmes — ajouter un sous-agent de vision
- (idea by coach) Schéma d'ingestion corpus platform-agnostique : même structure JSON pour Twitter, Facebook, TikTok, Reddit — abstraction des sources
- (idea by coach) Tracker de contenu supprimé : logger ce qui a été effacé et quand — l'absence est un signal, ne pas l'ignorer
- (idea by coach) Scorer de suspicion bot multi-dimensionnel : âge du compte + ratio followers/following + vitesse de posting + répétition verbatim
- (idea by coach) Classifier de type de crise en entrée du pipeline : réputation / attaque coordonnée / controverse organique / désinformation — chaque type oriente le pipeline
- (idea by coach) Bundle de preuves de coordination : exporter IDs de tweets + timestamps + phrasing identique comme pièces justificatives, pas juste une assertion
- (idea by coach) Détecteur de contre-narratif : identifier les voix qui ont résisté au cadrage dominant et mesurer leur reach vs le flux majoritaire
- (idea by coach) Tracker d'amplification médiatique : à quelle heure les médias traditionnels ont-ils relayé, et comment le framing a-t-il changé ?
- (idea by coach) Modèle de physique de propagation platform-spécifique : QTs vs reposts vs replies ont des dynamiques différentes selon la plateforme
- (idea by coach) Moniteur de glissement lexical : tracer comment le vocabulaire évolue (ex: 'polémique' → 'scandale' → 'épuration') heure par heure
- (idea by coach) Machine à états du cycle de crise : Étincelle → Ignition → Viralité → Pic → Réponse institutionnelle → Retombée — chaque état active des agents dédiés
- (idea by coach) Agent vérificateur cross-agent : re-lit les outputs de tous les agents, croise avec les données brutes, émet un score de confiance + liste des claims douteux
- (technique) started Job to Be Done — quel est le vrai job que l'outil est embauché pour faire ?
- (idea by coach) Job 'MAINTENANT QUOI ?' (institution en crise) → construire une file de priorité d'actions : faire X d'abord, puis Y, jamais Z — le système trie l'urgence
- (idea by coach) Job 'récit défendable' (chercheur/analyste) → construire un ledger de faits immuables avec citations avant toute interprétation — l'ordre compte
- (idea by coach) Job 'alerte précoce' → seuil d'alerte sur la vélocité d'engagement : quand l'accélération dépasse N× la baseline, déclencher le pipeline
- (idea by coach) Job 'comportement coordonné' → ne pas juste détecter, estimer la TAILLE du groupe coordonné et son reach effectif (combien de vues)
- (idea by coach) Job 'démo Datathon' → sortie visuelle narrative convaincante, pas juste du JSON backend — le keepsake HTML est le livrable visible
- (idea by coach) Job 'décideur politique' → brief exécutif d'une page avec scores de confiance, pas rapport de 40 pages — adapter le niveau de sortie au destinataire
- (idea by coach) Job 'multi-institution' → l'outil est configuré avec le contexte/mandat propre à chaque institution — paramétrable, pas hard-codé
- (idea by coach) Job 'vérification journalistique' → exporter des bundles de preuves qui satisfont aux standards journalistiques — traçabilité complète
- (idea by coach) Job 'monitoring temps-réel' → séparer le pipeline de surveillance (basse latence) du pipeline d'analyse profonde (haute qualité) — deux vitesses
- (idea by coach) Job 'communication institutionnelle' → l'agent rédacteur génère 3 versions de réponse : ton prudent / équilibré / assertif — le client choisit
- (idea by coach) Job 'apprentissage post-crise' → chaque crise traitée génère un doc de leçons apprises réinjecté dans la base de connaissance du système
- (idea by coach) Job 'prévention pas seulement réponse' → modéliser quels signaux précoces auraient permis d'éviter l'escalade — tourner le système vers l'avant
- (idea by coach) Job 'multi-langue' → une crise en français amplifiée en anglais nécessite une analyse bilingue simultanée — ne pas traiter les langues en silo
- (idea by coach) Job 'passage à l'échelle' → les agents fonctionnent sur 1 000 tweets ou 1 million avec la même interface — le volume est un paramètre
- (idea by coach) Job 'piste d'audit' → chaque décision du système est loggée et explicable à des parties prenantes non-techniques — observabilité de l'IA
