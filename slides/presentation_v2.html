Problématique

Analyser et répondre à l'échelle

Comment une institution peut-elle, face à une tempête virale, (1) détecter automatiquement la dynamique et les narratifs, (2) valider humainement avant d'agir, et (3) générer des options de réponse prêtes à l'emploi — de façon reproductible et neutre politiquement ?

❌ Avant
  • Veille manuelle, lente, incomplète
  • Détection subjective des narratifs
  • Rédaction de communiqués artisanale
  • Biais humain dans la priorisation
  • Non reproductible d'une crise à l'autre
✓ Avec notre pipeline
  • Analyse du corpus complet en quelques minutes
  • Classification LLM structurée + garde-fous pandas
  • 3 options de réponse en 3 tonalités
  • Charte de neutralité encodée dans les prompts
  • Générique : 1 paramètre change pour une nouvelle crise

— Comment classifier les narratifs à l'échelle ?
Agent · 1/4

Agent Analyste 🔍

📥 Input
  • raw_df — corpus complet (35 396 tweets)
  • corpus_config — événement, période, taxonomies injectées
Échantillon stratifié — allocation égale par type d'engagement (ORIGINAL / REPLY / QUOTE / RETWEET), quel que soit le corpus. Surreprésente les tweets originaux, plus riches en signal narratif.
📤 Output
  • narratives → résultat structuré
  • Narratif dominant + répartition (calculés en Python, pas par le LLM)
  • 1 analyse par tweet : narratif × type d'acteur
✓ Taxonomie entièrement configurable via corpus_config — aucun narratif hardcodé.
🤖 Réglages LLM
Réponses structurées — le LLM retourne directement un JSON typé, pas du texte libre. Impossible de sortir du format.
Température 0.2 = déterministe et factuel. Format tweet enrichi : [postID] TYPE|v:VERIFIED|f:FOLLOWERS|texte
batchs de 25 few-shot configurable
🎯 Mission
Classifier chaque tweet par narratif et type d'acteur — catégories injectées via corpus_config, adaptables à toute crise. Ex. Ultia : censure · copinage · défense_ultia · défense_cnc · autre — acteurs : média · militant · influenceur · anonyme · institution · coordonne_bot
Répond à : "Quels narratifs dominent la conversation, et qui les porte ?"
— Comment détecter une crise automatiquement ?
Agent · 2/4

Agent Veille 📡

📥 Input
  • raw_df — corpus complet (35 396 lignes)
Pourquoi le corpus complet ? Un échantillon manquerait les pics de volume et les signaux de coordination — qui sont précisément les indicateurs d'une crise.
📤 Output
  • Niveau d'alerte : low / medium / high / critical
  • Dates et volumes des pics détectés
  • Résumé de la dynamique (seule partie rédigée par le LLM)
🛡️ Principe fort : pandas d'abord
Tous les chiffres sont calculés en Python. Le LLM rédige uniquement le résumé en langage naturel — il ne peut pas halluciner de statistiques.
Seuils adaptatifs : le pic "normal" est calculé sur l'intervalle de confiance du volume du corpus (pas une valeur fixe codée en dur).
3 signaux de coordination : publications simultanées de groupes · séquences rapides d'un même compte · texte identique entre plusieurs comptes.
2 signaux acteurs : burst d'influenceurs (≥ N influenceurs distincts / jour) · cascade sentiment négatif chez les vérifiés.
🎯 Mission
Détecter pics de volume, tweets viraux et signaux de coordination — et calculer un niveau d'alerte objectif, jamais laissé à l'appréciation du LLM.
Répond à : "Y a-t-il une crise ? De quelle intensité ? Y a-t-il orchestration ?"
— Comment proposer des options de réponse institutionnelle ?
Agent · 3/4

Agent Stratège 🧭

📥 Input
  • Narratif dominant + répartition (sortie Analyste)
  • Niveau d'alerte + synthèse (sortie Veille)
  • Contexte de l'événement
  • Bloqué tant que l'équipe n'a pas validé via le HumanGate
📤 Output
  • 3 options : prudent · équilibré · assertif
  • Risques explicites pour chaque option
  • Option recommandée + justification
⚖️ Neutralité garantie
Le Stratège ne prend pas position sur le fond de la crise — il structure des options pour que l'équipe décide.
Chaque option expose ses avantages et ses risques avec la même rigueur. Pas de posture idéologique, pas de prise de partie.
🎯 Mission
Transformer l'analyse de crise en 3 options d'action concrètes, avec les risques associés, pour que la cellule de crise puisse décider en connaissance de cause.
Répond à : "Comment la cellule de crise devrait-elle répondre ?"
— Comment rédiger un communiqué en 3 tonalités ?
Agent · 4/4

Agent Rédacteur ✍️

📥 Input
  • 3 options stratégiques validées (sortie Stratège)
  • Narratif dominant à adresser
  • Contexte de l'événement
📤 Output
  • 3 communiqués : prudent · équilibré · assertif
  • Titre + corps + call-to-action par version
  • Tonalité recommandée parmi les 3
📋 Garanties éditoriales
Chaque affirmation du communiqué doit citer les identifiants des tweets sources qui la justifient — aucun claim sans preuve.
Pas de position politique sur le fond. Les 3 tonalités permettent à l'équipe de choisir la posture finale sans que le pipeline l'impose.
🎯 Mission
Rédiger un communiqué institutionnel factuel, prêt à publier, en 3 tonalités — en laissant à l'équipe le choix final de la posture.
Répond à : "Quelle est la réponse officielle, prête à être relayée ?"

Argumentaires

Honnêteté sur les limites, clarté sur la roadmap

⚠ Limites actuelles
📊
Pipeline sur un échantillon de 500/35 396 tweets — contrainte quota API, pas de résultat sur corpus complet
🤖
LLM identique pour 4 agents — un modèle spécialisé par tâche (classification vs rédaction) donnerait de meilleurs résultats
⏸️
HumanGate synchrone — bloque le pipeline en attente ; inadapté à une réponse en temps réel sans système de notification
📝
La neutralité encodée dans les prompts est une déclaration d'intention — la vraie garantie, c'est l'obligation de citer les tweets sources pour chaque affirmation
✦ Optimisations possibles
🔔
HumanGate asynchrone — notification Slack/email dès l'alerte, pipeline reprend à la validation sans bloquer
📊
Passage au corpus complet — actuellement limité par quota API LLM ; batch processing asynchrone sur 35 396 tweets
🧠
Modèle spécialisé par agent — un modèle de classification léger pour l'Analyste, un modèle génératif pour le Rédacteur
🌍
Support multilingue — le pipeline est langue-agnostique, les prompts et taxonomies suffisent à couvrir une crise internationale
🔄
Architecture générique opérationnelle — un seul corpus_config reconfigure l'ensemble pour toute nouvelle crise
